說明與教學

帶您快速了解系統原理與完整操作流程

🔍 系統原理：這是什麼工具？

想像系統是一個「自動化巡邏員（爬蟲）」，您給他一個起始網頁後，他會幫您點擊網頁上的每一個連結，一層一層地探索整個網站。

這套系統的特殊使命是：找出所有連到「外部網站」的連結，並檢查它們有沒有壞掉。

  • 防範失效連結： 幫您抓出網頁上已經變成「找不到網頁 (404)」的外部連結，避免使用者點擊後看到錯誤畫面。
  • 防範資安風險： 幫您揪出已過期失效的「無主網域」，讓您在被駭客搶註並植入惡意內容前及早移除。
  • 檢查傳輸安全： 幫您揪出哪些外部連結還在使用不安全的明文 HTTP 傳輸（非 HTTPS）。
  • 彙整不重複連結： 幫您將相同目標的網址自動合併，列出所有不重複的外部連結清單，以利您逐一檢視與清查。

📖 核心名詞解釋

在開始新增任務前，請先了解以下三個最重要的概念：

📍 起始 URL (Start URL)
這是巡邏員出發的第一個網頁（例如您的官網首頁）。他會從這裡開始蒐集連結。
🏠 目標網域 (Target Domains)
這是您劃定的「巡邏範圍（圍牆）」。爬蟲只會在這個網域內不斷點擊與深入，不會跑到別人的網站去繼續往下爬，以免爬到整個網際網路。
🌍 信任網域 (Trusted Domains)
這是用來判定安不安全的「檢查哨」。只要網址不在這個網域內，系統就會把它當作外部連結，並對它敲門測試（檢查存活與安全），但不會走進去繼續往下爬。

🚀 任務操作步驟

  1. 新增任務：點擊左側導覽列的「新增任務」。
  2. 填寫網址與範圍：貼上您要掃描的「起始 URL」。系統通常會自動幫您預填「目標網域」及「信任網域」。但您可以按照自身狀況調整，例如：在「目標網域」新增子網站、在「信任網域」新增 gov.tw 網域。
  3. 調整進階設定（可選）：您可以根據需求填寫進階參數：
    • 忽略的副檔名：系統已預設數十種不需要檢查的檔案副檔名（例如 .pdf 或 .zip），你可以再新增特定需求。
    • 忽略的路徑規則 (Regex)：讓爬蟲跳過不需要檢查的特定網址。例如：下載附件的特定網址型式。
    • 最大爬取深度：限制爬蟲最多只能點進去幾層。留空代表不管多深都爬。
    • 最大抓取頁數：限制爬蟲最多只能抓幾個網頁，避免大型網站掃描太久。
    • 請求延遲：爬蟲點擊下一頁前要等待的秒數，避免您的網站伺服器被爬蟲點掛（建議維持預設值）。
    • 連線逾時：爬蟲等待目標網站回應的最長秒數，超過時間將判定為連線失敗。
    • 失敗重試次數：遇到暫時性連線異常時系統自動重試的次數。
    💡 測試提醒：如果是進行測試，請務必設定「最大爬取深度」及「最大抓取頁數」，以節省時間並避免對伺服器產生壓力。
  4. 建立並啟動：點擊「建立任務並開始」，畫面會跳轉至任務詳情，您可以即時看到進度條與發現的外連數量正在增加。
  5. 排除指定網域（可選）：在任務詳情的結果列表中，您可以點選搜尋列旁的「設定排除網域」按鈕，將不需要關注的網域（如社群平台）隱藏並排除於報表之外。
  6. 接收通知與匯出報表：任務完成後，系統會自動發送通知信至您的信箱。您可以在下方「外部連結結果」列表查看失效的連結，或是將結果下載至電腦查看。

⚡ 系統效能與優化機制

面對擁有大量網頁與連結的網站，本系統內建了多項優化機制，在不影響目標伺服器正常運作的前提下，提升掃描效能與準確率：

  • 外連探測快取與並發： 若相同的外部連結重複出現，系統會利用「快取機制」紀錄前次結果，避免重複發送請求。同時採用多工並發技術，有效縮短整體檢驗時間。
  • MIME 類型檢查： 系統在發送請求時，會先檢查目標的檔案資訊（標頭）。若判斷為影音或壓縮檔等非網頁檔案，會主動中斷連線，節省網路頻寬與處理時間。
  • 動態降級讀取技術： 部分大型社群平台（如 Facebook, Instagram）會阻擋自動化探測程式。本系統可自動降級為微量讀取模式，減少外部連結被誤判為失效的機率。
  • 自訂信任網域： 您可以將特定的關聯網域（如 gov.tw 或子公司網址）加入「信任網域」，系統會將其視為內部連結並略過外部存活探測，節省不必要的檢查時間。
  • 彈性的資源控制： 系統支援「最大抓取頁數」限制，並可自訂「忽略的路徑規則」與「忽略的副檔名」，讓您能針對重點頁面進行掃描，避開不需要檢查的歷史或靜態資源區塊。

💡 常見問題 (FAQ)

Q: 報表中的「DNS 解析失敗」與「HTTP 錯誤」有什麼不同？

DNS 解析失敗：代表這個網址的主機名稱已經完全找不到了（通常是網域沒繳錢過期了，或是打錯字）。這屬於高風險，因為駭客隨時可以把這個無主網域買下來，放上惡意內容。
HTTP 錯誤：代表主機還在，但網頁內容不見了或伺服器當機（常見的狀態碼有 404 找不到網頁、500 伺服器錯誤）。

Q: 為什麼任務執行了很久都沒結束？

為了不讓爬蟲把您的網站伺服器塞爆，系統預設在每次點擊之間都會有「請求延遲」（例如 3 秒）。如果您的網站有幾千頁，加總起來就會需要幾個小時才能掃描完。建議可以利用「最大抓取頁數」來限制掃描規模。

Q: 為什麼有些正常的外部連結（例如 Facebook）會顯示為 HTTP 錯誤？

許多大型網站（如 Facebook 等社群平台）設有嚴格的防爬蟲機制。當系統自動檢查這些連結時，可能會被對方阻擋並回傳 HTTP 錯誤（例如 403 禁止存取）。雖然系統會嘗試繞過防護，但仍可能會出現大量的錯誤回報，這屬於正常現象。

Q: 任務卡在「執行中」或是發生未知的錯誤怎麼辦？

您可以點擊任務詳情中的「暫停」按鈕，並在暫停後點擊「重置」來重新開始。如果任務徹底卡死，請聯絡系統管理員從後台進行「強制接管」。
